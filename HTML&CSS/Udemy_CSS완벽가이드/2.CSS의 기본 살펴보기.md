## 21. 이론 - 결합자
- Adjacent Sibling: `div + p {...}`
  * `div` 다음에 나오는 `p` 태그에만 적용
- General Sibling: `div ~ p {...}`
  * `div` 와 같은 부모를 공유하는 `p`태그에 적용
- Child: `div > p {...}`
  * `div`의 직속 자식 `p`에만 적용
- Descendant: `div p {...}`
  * `div`의 직속 자식이 아니라도 `div`의 자손이기만 하면 적용
  * 가장 자주 사용

### 선택자
-`=`: 완전 일치 `[type="text"]`
-`*=`: 포함(부분 일치) `[class*="btn"]`
-`^=`: 앞에서부터 일치 `[class^="btn-"]`
-`$=`: 뒤에서 부터 일치(끝) `[class$="-error"]`
-`~=`: 띄어쓰기 기준 단어 포함 `[class~="active"]`
  * `<div class="btn primary active large">` -> O
  * `<div class="btn-incative">` -> X
- `|=`: 하이픈(-)으로 구분된 접두사(prefix) 메칭 `[class|=val]`
  * attr 값이 "val" 이거나 "val-" 로 시작하면 선택

- `:is()`: 여러 선택자 중 하나라도 매칭되면 OK
  * `:is(h1, h2, h3)`
  * `.card :is(a, button)`
- `:not()`: 괄호 안에 해당하는 건 제외
  * `.btn:not(.btn--ghost)`
  * `input:not([type="button"]`
  * `.item:not(.hidden, .disabled)`
- `:where()`: `:is()`랑 비슷한데 명시도 0(우선순위 낮게)
  * `:is()`와 거의 똑같음.
  * 기본 스타일(리셋/노말라이즈) 만들 때 유용
- `:hss()`: 부모선택자
  * `.field:has(.error)` : 자식 중에 .error가 하나라도 있는 .field 만
- `nth-child()`: 자식 선택자
  * `li:nth-child(3)`: 3번째 자식
- `nth-of-type()`: 타입기준
  * `p:nth-of-type(2)` 같은 타입 기준 두번째 `<p>`
