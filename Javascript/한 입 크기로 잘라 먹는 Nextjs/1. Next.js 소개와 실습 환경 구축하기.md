# 1. Next.js 소개와 실습 환경 구축하기
## Next.js를 소개 합니다
### 리액트의 확장판인 Next.js
- Next.js는 Vercel이라는 미국의 스타트업이 만든 웹 개발용 메타 프레임워크입니다. 메타 프레임워크란 다른 라이브러리나 프레임워크를 확장해 주는 프레임 워크라고 말할 수 있습니다.
- Next.js는 리액트를 포함하는 메타 프레임워크로서 리액트의 훌륭한 UI 개발 기능에 더해 페이지 라우팅, 서버 사이드 렌더링 등과 같은 기능을 추가로 제공합니다.
- 서버사이드 렌더링이 Next.js의 핵심 기능임은 분명하지만, 이 기능 외에도 Next.js는 라우팅, 최적화, 스타일링등 웹 서비스 개발에 필요한 기능을 종합적으로 제공합니다.

## 실습환경 설정하기
### Node.js
```shell
# nvm 업그레이드
brew update
brew upgrade nvm
nvm -v

# node lts 설치
nvm install --lts
node -v
npm -v
```

### 비주얼 스튜디어 코드
- Prettier: 코드 포멧팅 도구
- ESLint: 코드 품질 정적 검사 도구
- Error Lens: 오류 메시지를 더 직관적으로 표시해 주는 도구

### 실습용 백엔드 서버 설정하기
#### 1.백엔드 서버 코드 다운로드 받기
- `https://github.com/onebite-nextjs/book__onebite-books-server` -> Code -> Download ZIP

#### 2. Supabase 계정 생성하기
- Supabase는 복잡한 설정 없이도 손쉽게 데이터베이스르 생성하고 이용할 수 있는 클라우드 기반 데이터베이스 서비스 입니다.
- `https://supabase.com/` 

#### 3. 백엔드 서버와 Supabase를 연결하는 커넥션 스트링 설정하기
- Connect -> ORMs 탭 이동 -> 환경파일 복사 -> 예제 루트아래 .env 파일을 생성 -> 비밀번호 입력

#### 4. 데이터 베이스 초기화하기
- 예제 프로젝트 `npm install`
- `npx prisma db push`
  * 설정값: `prisma/schema.prisma`
- supabase 로 돌아가 왼쪽 Table Editor 클릭하여 테이블이 생성되었는지 확인

#### 5. 시드 데이터 삽입하기
- `npm run seed`

#### 6. 백엔드 서버 가동하기
- `npm run build`
- `npm run start`
- `http://localhost:8080`

#### 7. 프리즈마 스튜디어 사용하기
- 백엔드 서버를 사용하는 데이터베이스의 데이터를 훨씬 더 편리하게 조회하고 관리하는 두고
- Prisma Studio는 현재 연결된 데이터베이스의 데이터를 편리하게 조회, 수정, 삭제할 수 있는 Prisma가 제공하는 GUI도구 입니다.
- 새터미털 추가 -> `npx prisma studio`

#### 실습용 백엔드 서버를 사용할 때 주의할 사항
- Supabase는 앞서 만든 데이터베이스에 약 7일간 아무런 접근도 하지 않으면 자동으로 프로젝트를 중단합니다.
- 대시보드에 **Project is paused** 라는 메시지가 표시 되어있다면 프로젝트가 지금 중단 상태라는 의미
- **Restore project** 버튼 클릭 시 프로젝트 가동 할 수 있음
